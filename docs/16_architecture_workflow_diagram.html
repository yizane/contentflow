ContentFlow 当前工作流图（Phase 13）

本文档只记录当前仓库真实运行的 workflow。运行态数据只进 MySQL；本地文件只保留代码、配置、prompt、schema、迁移和文档。

engine:daily 负责 daily 幂等和运行控制，实际主链路由 engine_batch.js 串行执行。OpenClaw 只作为模型/智能体执行层；状态、正文、评分、trace、决策解释都以 MySQL 为准。

发布包、人工终审、报告、历史分类回填、选题压力测试是后续人工/运维动作，不属于 daily 主编排的必经步骤。

engine_batch 主链路

顺序 节点 入口/脚本 类型 主要写入 要点
1 采集来源 + 内容分类 pipeline/sources_collect.js 规则 + OpenClaw source_items、source_collection_logs、content_classifications HTTP/RSS 规则采集；search_query 调 OpenClaw；分类规则优先，低置信调用 AI。
2 生成主题池 pipeline/topics_generate.js OpenClaw topic_candidates、model_runs 候选必须包含 raw score、content value score、分类字段；缺失按软着陆规则回退。
3 组合选题并创建 job pipeline/jobs_create.js 规则 article_jobs、topic_candidates.selection_*、status_transitions 高分但饱和/重复/来源弱进入 deferred；低价值不应伪装成 rejected。
4 生成文章 + 视觉规划 pipeline/jobs_run.js OpenClaw articles、article_versions、quality_reports、model_runs 文章 schema 要求视觉规划；正文禁止 AI/联网失败痕迹；质量自检不是终审门禁。
5 事实核查 + 主质量门禁 pipeline/factcheck_run.js OpenClaw + 门禁 fact_checks、article_quality_scores、articles.status article_quality_score < 80 进入 needs_quality_revision；事实缺源进入 needs_fact_sources。
6 渠道改写 pipeline/channels_generate.js OpenClaw channel_outputs、model_runs 生成公众号、抖音、小红书稿；这是内容资产，不是真实发布。
7 SEO/GEO 评分 pipeline/score_seo_geo.js OpenClaw seo_geo_scores、文章/版本分数字段 只处理 ready_for_review；SEO/GEO 是建议线，不能覆盖主质量分和事实底线。
8 收尾快照 tools/db_list.js + engine_batch.js 规则 engine_runs.summary_json、workflow_steps 打印当前文章快照，汇总计数、错误、warning、traceFailures。

主链路之外的人工/运维动作

命令 类型 作用 主要写入
npm run sources:fix 闭环修订 处理 needs_fact_sources：来源补全、局部修订、再次事实核查。 source_resolutions、新版 article_versions、fact_checks
npm run score:article-quality 质量复评 对存量或指定文章重跑主质量评分；可用于验证修订是否过 80。 article_quality_scores、文章/版本主评分字段
npm run topic:audition 选题压力测试 模拟未来 N 轮选题，不生成文章，用于发现重复、偏科和低价值候选。 topic_audition_runs、topic_audition_items
npm run content:classify 回填 + OpenClaw 给历史 source/topic/article 回填三层内容分类。 实体分类字段、content_classifications
npm run package:export 发布包生成 把正文、质量、事实、渠道、视觉规划整合成可被 Web 读取的包；不落 output 文件。 publish_packages
npm run review:mark 人工终审 人工标记 reviewed / rejected / approved_for_publish / archived；低于 80 的主评分不能通过。 review_actions、status_transitions
npm run engine:report 运维报告 生成生产报告，包含 qualityOverview、portfolioHealth、taxonomySummary。 engine_reports
