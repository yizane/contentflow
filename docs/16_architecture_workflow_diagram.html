ContentFlow Python Workflow 图

当前 workflow runtime 是 workflow_py/contentflow。workflow 不保留 root npm/Node 兼容入口；Viewer/Web 通过 uv run contentflow ... 触发 Python workflow。

运行态数据只进 MySQL。本地文件只保留代码、配置、prompt、schema、migration 和文档。

2. Python 模块职责

flow/

batch/daily 编排、补位循环、run control、step registry、LangGraph 图。

domains/sources/

采集、AMZ123 API、正文抽取、canonical ingest。

domains/topics/

选题生成、来源守门、去重、选题压力测试。

domains/production/

文章生成、事实核查、补来源、质量门、渠道、发布包、终审。

llm/

prompt、schema 校验、模型调用审计、OpenClaw provider。

commands/ + ops/

CLI 命令分组、运行报告、配置同步、运维检查。

3. 调试入口

  • uv run contentflow engine batch --limit 1 --dry-run：验证参数和主链路计划。
  • uv run contentflow topic audition --rounds 10 --limit 3：模拟候选池能否持续产出。
  • uv run contentflow sources check：检查来源配置、lane 和采集健康。
  • uv run pytest：运行 Python pytest。
